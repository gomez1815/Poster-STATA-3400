The purpose of this lesson is to estimate and interpret a multiple linear regression model with a categorical predictor (explanatory) variable.

Indicator Formulation

In our previous discussions of multiple linear regression the response and predictors were both continuous, numerical variables. Instead, we might like to use a categorical variable to predict a numerical response. However, this requires a slightly different approach since the predictor categories are generally represented by words rather than numbers. How do we write a regression equation with words?! The solution is to treat each category (level) as its own binary variable that is equal to 1 if the observation is in the category and 0 if it is not in the category. We call these artificial predictors indicator variables because they indicate which category is assigned.

Formally, if a predictor variable \(z\) has values consisting of \(k\) categories then the linear regression equation is defined below.

\[ y=\beta_0 +\beta_1 z_1 +\beta_2 z_2 + ...+ \beta_{k-1} z_{k-1} + \epsilon \]

In this equation, \(z_i\) is equal to 1 if an observation is assigned a \(z\) value equivalent to category \(i\) and 0 otherwise. Furthermore, since an observation can only be assigned a single category, it must be true that:

\[ z_1 + z_2 + ... + z_{k-1} \leq 1 \]

There are two important items to note in this formulation. First, we only need to define \(k-1\) indicator variables even though there are \(k\) categories. This is because we use the intercept \(\beta_0\) as a reference level. In other words, if an observation belongs to the \(k\)th category, then all of the \(z_i\) values will be equal to 0 and only the intercept will remain. This also explains why we use an inequality in the constraint for the sum of the \(z_i\) values. If an observation is in the \(k\)th category, then the sum of the \(z_i\) values will be 0. Otherwise, the sum must be exactly equal to 1.

Inferential Model

Suppose we want to see if there is an association between the species of penguin (categorical) and the length of its flippers (numerical).

ggplot(data = penguins2, aes(x = species, y = flipper_length_mm)) +
  geom_boxplot(fill = "#99FFFF") +
  labs(
    x = "Penguin Species",
    y = "Flipper Length (mm)"
  ) +
  theme_bw()

Based on the box plot, it does appear there is some association between species and flipper length. The predictor variable has three categories (species). So, based on our previous formulation, the linear regression equation only requires two indicator variables. One of the species will be left out as the reference category. In that case, the estimated regression equation would be:

\[ \hat{y}=\hat{\beta}_0 + \hat{\beta}_1 \cdot \text{Chinstrap} + \hat{\beta}_2 \cdot \text{Gentoo} \]

In this equation \(\text{Chinstrap}\) and \(\text{Gentoo}\) are binary indicator variables that add to at most 1. Just as with our previous discussion of regression equations, the \(\hat{\beta}\) parameters are what we seek to estimate using the least squares method.

# fit model
model1 <- lm(flipper_length_mm ~ species, data = penguins2)

# view estimated parameters

The resulting estimated regression equation is:

\[ \hat{y}=190.10 + 5.72 \cdot \text{Chinstrap} + 27.13 \cdot \text{Gentoo} \]

Now let’s carefully interpret the meaning of this estimated regression equation. If a given penguin is an Adelie, then the Chinstrap and Gentoo predictors are both equal to zero and we estimate Adelies have an average flipper length of 190.10 mm. Prediction Model

Now that we’ve determined the quality of the inferential model, we might like to use species to predict flipper length. In order to assess the accuracy of such a model, we first split the sample into training and testing sets.

# randomly select training rows
set.seed(303)
rows <- sample(x = nrow(penguins2), size = floor(0.75 * nrow(penguins2)), replace = FALSE)

# extract training and testing sets
training <- penguins2[rows, ]
testing <- penguins2[-rows, ]

Now we fit the model to the training data and assess its accuracy using the testing data.

# fit model to training set
model2 <- lm(flipper_length_mm ~ species, data = training)

# predict testing set
testing_pred <- testing %>%
  mutate(flipper_pred = predict(model2, newdata = testing))

# compute RMSE
testing_pred %>%
  summarize(RMSE = sqrt(mean((flipper_length_mm - flipper_pred)^2)))

By only using species, we can predict a penguin’s flipper length to within about 7 mm. How does this compare to the null model that always predicts the mean flipper length?

# compute mean response
flipper_avg <- mean(training$flipper_length_mm)

# compute RMSE for null model
testing %>%
  summarize(RMSE = sqrt(mean((flipper_length_mm - flipper_avg)^2)))

Our species model performs much better than the null model. But, could we produce an even more accurate model by adding another predictor?

pairs(flipper_length_mm ~ ., data = training)

Let’s try adding body mass to the model.

# fit model to training set
model3 <- lm(flipper_length_mm ~ species + body_mass_g, data = training)

# predict testing set
testing %>%
  mutate(flipper_pred = predict(model3, newdata = testing)) %>%
  summarize(RMSE = sqrt(mean((flipper_length_mm - flipper_pred)^2)))

With the additional information of body mass, we can decrease the error even further to about 5.7 mm. What is the form of this new model?
